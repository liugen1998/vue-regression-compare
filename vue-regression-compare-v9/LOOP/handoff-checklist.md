# Agent 交接清单

最终回复或跨轮交接前检查：

- [ ] 新请求优先于旧上下文。
- [ ] 已说明实际完成结果，而不是只描述计划。
- [ ] 已列出变更文件。
- [ ] 已列出 Harness 级别和验证命令。
- [ ] 已说明未验证项和原因。
- [ ] 已说明服务地址、端口或生成产物位置。
- [ ] 已提醒敏感数据或登录态风险。
- [ ] 没有要求用户重复提供已在上下文中的服务器、账号、路径信息。

推荐交接句式：

```text
已完成 X。改动集中在 A、B。验证使用 H3 UI，运行了 typecheck、validate-config 和 /api/health。未跑 H4，因为 sample-page URL 是占位地址。后续如果接真实页面，优先补 workspace/pages/<page>.yaml 的稳定 selector。
```
